# DriveHelper

Telegram-бот для записи на занятия в автошколе. Ученики регистрируются, бронируют
свободные слоты, отменяют и переносят записи. Преподаватели создают слоты. Расписание
показывается по неделям всем. Администратор управляет пользователями (роли и удаление).

## Возможности

**Ученик (student)**
- Регистрация по `/start` — только ФИО (телефон не запрашивается).
- «Записаться» — выбор дня недели (кнопки с днями, где есть свободные слоты), затем
  времени из свободных слотов. Неделя листается кнопками ‹ / ›. Выходные (сб/вс)
  предлагаются только если включены в «Настройках».
- Выбор напоминания (за 30 мин / 1 / 2 / 4 / 8 часов / сутки / без напоминания).
- «Мои записи» — просмотр, отмена и перенос активных записей.

**Расписание (все роли)**
- «Расписание» — слоты на неделю, сгруппированные по дням. Свободные слоты
  показываются как «свободно», занятые — «занято: ФИО записавшегося». Неделя
  листается кнопками ‹ / › (назад — не раньше текущей недели). Доступно всем
  зарегистрированным пользователям. Выходные (сб/вс) показываются только если
  включены в «Настройках».

**Преподаватель (teacher)**

Управление слотами полностью кнопочное (тап по времени переключает ✅/⬜):
- «Ведение расписания» — недельный редактор. Показывает каждый видимый день
  недели с его слотами (свободно / занято: ФИО / нет слотов). Кнопка дня
  открывает сетку времён, предзаполненную текущими слотами дня; «Готово» делает
  слоты дня ровно отмеченными — новые времена создаются, снятые свободные
  удаляются, прошедшие времена пропускаются. Если снять время с активной бронью,
  бронь отменяется: слот освобождается и удаляется, напоминание снимается, а
  записанный ученик получает прямое уведомление об отмене. Перед этим показывается
  подтверждение со списком отменяемых броней (кнопки «Подтвердить отмену» /
  «Отмена») — без снятых броней изменения применяются сразу. Кнопка «Задать
  времена на всю неделю» применяет один набор ко всем видимым дням недели сразу
  (её подтверждение агрегирует отменяемые брони по всем дням). Неделя листается
  кнопками ‹ / › (вперёд — в том числе на пустые будущие недели, чтобы их заполнить).
- «Управление записями» — недельный обзор слотов по дням. Тап по дню открывает
  его слоты (время + статус), тап по слоту позволяет:
  - освободить занятый слот — бронь отменяется (без проверки владельца),
    слот освобождается, напоминание снимается, а записанный ученик получает
    уведомление об отмене;
  - записать выбранного из инлайн-списка (по ФИО) пользователя на свободный
    слот — запись атомарна (без гонки с самозаписью ученика), без напоминания;
    ученик получает уведомление о записи.
  Показываются все дни, где есть слоты, включая выходные (независимо от
  настройки видимости выходных), чтобы любая бронь оставалась управляемой.
- «Времена по умолчанию» — сохранить пресет времён, который подставляется в
  «Задать времена на всю неделю».
- «Настройки» — показывать ли выходные (сб/вс) в расписании, записи и редакторе.
  По умолчанию выходные скрыты; переключается одной кнопкой.

**Администратор (admin)**
- Все возможности преподавателя.
- «Пользователи» — единое меню управления пользователями (инлайн, по ФИО, без
  ввода Telegram id). Список всех зарегистрированных пользователей с их ролью;
  тап по пользователю открывает карточку, где можно:
  - сменить роль — назначить ученика преподавателем или снять преподавателя;
  - удалить пользователя (с подтверждением) — его будущие записи освобождаются
    (слоты снова доступны для брони, напоминания снимаются), созданные им слоты
    сохраняются. Администратора (из `ADMIN_IDS`) удалить или сменить ему роль
    нельзя. Администратор — суперпользователь.

## Роли

- `student` — по умолчанию при регистрации.
- `teacher` — назначается администратором из инлайн-списка.
- `admin` — Telegram id из переменной окружения `ADMIN_IDS` (это источник истины).
  Если id есть в `ADMIN_IDS`, роль `admin` также проставляется в БД при первом
  контакте / регистрации.

## Часовой пояс

Все даты хранятся в БД как наивный UTC, а отображаются и вводятся в часовом поясе
школы (`TZ`, по умолчанию `Europe/Minsk`). Границы недели (Пн 00:00 — следующий
Пн 00:00) для расписания и записи считаются в часовом поясе школы, поэтому слот в
23:00 UTC воскресенья попадает в правильный локальный день.

## Настройка

1. Скопируйте пример окружения и заполните значения:
   ```bash
   cp .env.example .env
   ```
2. Переменные:
   - `BOT_TOKEN` — токен бота (обязательно).
   - `ADMIN_IDS` — Telegram id администраторов через запятую, напр. `111111111,222222222`.
   - `TZ` — часовой пояс школы (IANA), по умолчанию `Europe/Minsk`.
   - `DEFAULT_SLOT_DURATION_MIN` — длительность слота в минутах (по умолчанию 90).
   - `DB_PATH` — путь к файлу SQLite (по умолчанию `/data/drivehelper.db`).
   - `SLOT_TIME_START` — начало диапазона времён в сетке пикера, `ЧЧ:ММ`
     (по умолчанию `07:00`).
   - `SLOT_TIME_END` — конец диапазона времён в сетке пикера, `ЧЧ:ММ`
     (по умолчанию `21:00`).
   - `SLOT_TIME_STEP_MIN` — шаг между временами в сетке в минутах
     (по умолчанию `30`).

### Как получить `BOT_TOKEN`

Напишите [@BotFather](https://t.me/BotFather), команда `/newbot`, следуйте
инструкциям — он выдаст токен вида `123456:ABC-DEF...`.

### Как узнать свой Telegram id для `ADMIN_IDS`

Напишите [@userinfobot](https://t.me/userinfobot) или [@getmyid_bot](https://t.me/getmyid_bot) —
они пришлют ваш числовой id.

## Запуск

```bash
docker compose up -d --build
```

База данных сохраняется в именованном томе `drivehelper-data`.

## Локальный запуск (без Docker)

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
BOT_TOKEN=... ADMIN_IDS=... DB_PATH=./drivehelper.db .venv/bin/python -m bot.main
```
